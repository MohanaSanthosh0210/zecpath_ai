# HR Interview Scoring Engine

## Overview

The HR Interview Scoring Engine combines outputs from multiple AI interview modules into a single normalized HR Interview Score.

Unlike standalone scoring systems, this engine consumes the outputs produced by previous stages of the AI interview pipeline.

---

# Pipeline

Candidate Audio

↓

Day 24 — Speech-to-Text

↓

Day 25 — Answer Understanding

↓

Day 35 — Communication Evaluation

↓

Day 36 — Behavioral Intelligence

↓

Day 37 — HR Interview Scoring

↓

HR Score Report

---

# Input Sources

The scoring engine reads the following outputs automatically:

| Module | Input File |
|---------|------------|
| Day 25 | data/understood_answers/sample_answer.json |
| Day 35 | data/communication/communication_score.json |
| Day 36 | data/behavioral_analysis/behavioral_report.json |

No manual answer input is required.

---

# Evaluation Components

The HR Interview Score combines four evaluation dimensions.

## Answer Relevance

Measures:

- Intent confidence
- Off-topic responses
- Missing information
- Response relevance

Generated by:

relevance_scorer.py

---

## Communication

Obtained directly from Day 35.

Measures:

- Fluency
- Grammar
- Vocabulary
- Clarity
- Answer Structure

---

## Confidence

Obtained directly from Day 36.

Measures:

- Hesitation
- Stress indicators
- Behavioral confidence

---

## Consistency

Measures response consistency using Day 25 structured answer analysis.

Generated by:

consistency_scorer.py

---

# Processing Flow

The engine performs the following steps:

1. Load previous module outputs.
2. Load configurable scoring weights.
3. Calculate Answer Relevance.
4. Calculate Consistency.
5. Combine Communication Score.
6. Combine Confidence Score.
7. Compute weighted HR Score.
8. Normalize interview score.
9. Send the result to the Report Generator.

---

# Output

The engine returns structured scoring data which is later converted into the HR Interview Report.
