# Question Hierarchy Model

Version: 2.0

---

# Objective

The Question Hierarchy Model defines how technical questions are selected and progressively increased in difficulty based on role and experience.

---

# Interview Hierarchy

Technical Interview

│

├── Introduction

├── Experience Discussion

├── Concept Questions

├── Practical Questions

├── Scenario-Based Questions

├── Technical Evaluation

└── Summary

---

# Experience Mapping

## Entry Level (0–2 Years)

Focus

- Programming Basics
- OOP
- Data Structures
- SQL
- Basic Problem Solving

Difficulty

Easy → Medium

---

## Intermediate (3–5 Years)

Focus

- APIs
- Frameworks
- Debugging
- Database Design
- Optimization

Difficulty

Easy → Medium → Hard

---

## Advanced (5+ Years)

Focus

- Architecture
- Scalability
- System Design
- Performance
- Distributed Systems

Difficulty

Medium → Hard → Expert
